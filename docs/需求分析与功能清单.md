# 农业资源数字化管理应用平台 — 需求分析与功能清单

> 来源：《二期数字农业与乡村管理平台 — 农业资源数字化管理应用平台》设计文档（章节 12.2.6.6）
> 本文档用于：梳理需实施的功能、功能层级与关系、功能细节，并标注实现要点（常见问题、可测试性、UI、数据）。

---

## 1. 项目概述

农业资源数字化管理应用平台是数字农业与乡村管理平台的数据应用组件，以确权信息、农业资源普查数据为支撑，构建农业"一张图"管理体系。整体设计理念为 **"一云、一库、一图、N 应用"**：

- **一云**：一个云计算中心
- **一库**：一个农业农村数据资源库
- **一图**：一张农业资源地理底图
- **N 应用**：覆盖耕地利用、资源管理、种植动态分析等多业务应用

平台面向 **政府监管** 与 **农业企业** 两类核心用户，提供差异化服务：
- 政府监管：撂荒监测、种植结构分析、耕地质量评价，辅助决策
- 农业企业：产地分布、作物种植空间分布等地图服务，支撑生产布局与市场分析

## 2. 技术栈与部署约束

| 层 | 技术 | 备注 |
|---|---|---|
| 数据库 | MySQL 5.7 | 注意 5.7 不支持 `JSON` 函数全集、不支持窗口函数、`utf8mb4` 默认需显式配置 |
| 后端 | Spring Boot + JDK 1.8 | Spring Boot 建议 2.7.x（兼容 JDK8 的最后主线）；MyBatis-Plus / JPA 任选 |
| 前端 | Vite + Vue 3 + Element Plus | 组合式 API，Pinia 状态管理，Vue Router |
| 部署 | 本地 Docker | docker-compose：mysql + backend + frontend(nginx) 三容器起步 |
| 地图（可选） | Leaflet / OpenLayers | 文档大量涉及 GIS，本地 MVP 阶段可用经纬度点 + 简单底图替代 |

**MySQL 5.7 关键注意点（影响建表）：**
- 字符集统一 `utf8mb4` / `utf8mb4_general_ci`，避免中文/emoji 报错。
- 空间数据：5.7 支持 `GEOMETRY` 类型与基础空间函数，但生产成熟度有限。MVP 建议地块边界以 GeoJSON 文本（`LONGTEXT`）存储 + 中心点 `lng/lat DECIMAL(10,6)` 字段，后续再迁移到 PostGIS。
- 无窗口函数 → 排名/累计类分析需在应用层或子查询实现。

**模块间核心关系（数据流）：**

```
确权地块(A) ──是数据底座，被以下模块通过 parcel_code 关联──┐
   ├──→ 种植记录(C1)：地块 1:N 年度种植记录
   ├──→ 耕地质量(C2)：地块 1:N 年度质量评价
   ├──→ 撂荒地块(D1)：地块 1:N 撂荒记录（支持部分关联，记录占比）
   └──→ 标注/历史(A5/A6)：地块 1:N 版本/标注
撂荒地块(D1) ──→ 撂荒原因(D2) ──→ 治理任务(D3) ──→ 回写种植记录(C1)
种植记录(C1) + 质量(C2) + 撂荒(D1) ──聚合──→ 种植动态分析(E)
水利/配套设施(B) ── 独立资源库，空间上叠加于一张图
```

## 4. 模块详细功能说明

> 每个叶子功能给出：**功能描述 / 关键字段与输入 / 业务规则 / 常见问题注意点 / 可测试点**。

### A. 土地资源数据

> 平台数据核心，确权数据准确性直接影响所有上层应用。**MVP 建议优先实现 A1.2/A1.3/A1.5/A1.6，A1.1 的文件导入与 A1.4 的 GIS 几何编辑作为进阶。**

#### A1.1 确权地块数据导入
- **描述**：批量导入农村土地承包经营权确权登记成果。支持 Shapefile/GeoJSON/KML/GML/Excel；4 步流程：上传 → 校验 → 空间处理 → 入库。
- **关键字段**：地块编码、承包方姓名/编码、确权面积、地块四至（东南西北）、承包起止日期、地块用途、行政区划。
- **业务规则**：
  - 校验：地块编码唯一性、坐标系校验、几何有效性（自相交/空洞）、必填字段完整性 → 生成校验报告并定位错误。
  - 空间处理：建空间索引、算地理面积并与确权面积比对（超阈值提示）、生成中心点、关联行政区划。
  - 入库模式：**增量导入** / **全量覆盖**。生成地块 ID 与数据质量报告（成功/失败数、各环节通过率）。
- **常见问题**：大文件（>100MB）分片上传与断点续传；编码重复导致覆盖；坐标系不一致（WGS84/CGCS2000/百度/高德偏移）；Excel 与矢量绑定失败。
- **可测试点**：单元测试覆盖校验器（重复编码/缺字段/面积超差/年份非法）；导入幂等性（增量重复导入不产生重复行）；质量报告数字正确性。

#### A1.2 确权地块数据更新
- **描述**：对已入库确权数据维护。支持 **单个编辑** 与 **批量更新**。
- **业务规则**：
  - 单编辑字段：承包方姓名/编码、确权面积（与地理面积差异提示）、承包起止、用途、备注；几何支持节点/修形/分割/合并。
  - 分割：自动给子地块生成新编码（后缀 -1/-2）；合并：选保留编码，另一作废并记历史。
  - 所有编辑写 **版本日志**（编辑人/时间/类型/前后对比/原因）；关键字段（编码、承包方编码）改动需变更说明 + 二次确认。
  - 批量：Excel 模板按编码匹配更新；或空间框选统一改属性。批量前提供预览（影响记录数 + 变更内容）。
- **常见问题**：批量更新误伤、缺乏预览/回滚；关键字段改动断裂关联关系；版本日志缺失导致不可追溯。
- **可测试点**：版本日志生成完整；批量预览数与实际更新数一致；二次确认拦截关键字段。

#### A1.3 地块属性查询
- **描述**：地图点选查询 + 属性条件查询。
- **业务规则**：
  - 点选：弹确权信息卡（编码、名称、坐落、面积、承包方、期限、四至、用途、最新种植作物/年度），"查看详情"进详情页，结果可导出 PDF/打印。
  - 条件：编码（模糊）、承包方姓名/编码、五级行政区划联动、面积范围、期限范围；组合为"且"；列表 + 定位按钮；导出 Excel/CSV。
- **常见问题**：模糊查询性能（编码、姓名加索引）；五级联动数据加载；导出大数据量内存溢出（流式导出）。
- **可测试点**：组合查询条件正确拼接；分页正确；导出行数与查询一致。

#### A1.4 地块编辑（GIS 几何编辑）
- **描述**：高精度空间几何编辑工具集（侧重几何，区别于 A1.2）。
- **工具**：节点编辑（增删改节点、实时坐标/线段长/面积对比）、边界修形（重绘替换段）、地块分割（绘分割线/按目标面积分割）、地块合并（相邻共边、保留编码、原地块标"已合并"并继承种植历史）。
- **业务规则**：每次提交弹确认窗，必填 **编辑原因** + 可传佐证材料（测量表/测绘报告/流转合同）。
- **常见问题**：拓扑校验（自相交、面积突变）；前端地图编辑库依赖；MVP 阶段成本高，可延后。
- **可测试点**：分割后子地块面积之和 ≈ 原面积；合并后编码唯一、历史保留。

#### A1.5 地块历史回溯
- **描述**：时间轴展示地块全部变更（形态/面积/属性 + 时间/人/原因/审批人）；支持年月缩放。
- **核心特色**：**版本对比**（并排两版本，差异高亮，联动地图，导出 PDF 对比报告）；按时间段统计（变更次数、面积累计变化、承包方变更，生成生命周期报告）。
- **可测试点**：任意两版本对比差异计算正确；时间段统计聚合正确。

#### A1.6 地块标注
- **描述**：用户在确权数据之上叠加自定义标记，不影响底层数据。
- **类型**：文字标注（字体/字号/颜色/是否默认显示）、颜色标记（预设：红高风险/黄关注/绿正常/蓝已处理，可按色筛选）、标签分类（高产田/示范田/流转田等，多标签）、附件上传（图片/PDF/Word/Excel）。
- **权限**：可见范围 = 仅自己 / 本部门 / 指定角色 / 所有人；图层可显隐叠加。
- **可测试点**：标注权限可见性过滤正确；按颜色/标签筛选正确。

### B. 农业资源数据管理

> 水利设施与配套设施两类资源数字化，列表/查询/标注/修改/删除五类操作高度同构，**可抽象为通用"设施"基类 + 类型差异字段**，减少重复代码、便于测试。

#### B1. 水利设施管理（机井/泵站/水闸/渠道/滴灌/喷灌/蓄水池）

**B1.1 水利设施列表**
- 字段：设施名称、类型、所在位置（五级）、经度、纬度、建设年份、覆盖面积（亩）、受益村组、运行状态（正常/维修中/废弃/待改造）、管护责任人、责任人电话、最近维护日期、备注。
- 规则：分页（默认 20，可 10/20/50/100）、任意列排序、顶部搜索筛选（名称模糊 + 类型多选 + 状态单选 + 区域五级 + 建设年份范围），筛选实时刷新显示数量，可保存"我的筛选方案"。行操作：详情/编辑/定位/删除，支持批量选择与批量删除。
- 注意：批量删除二次确认；列表性能（分页 + 索引）。

**B1.2 水利设施查询**
- 维度：空间范围查询（地图绘矩形/圆/多边形，按距离排序如"周边5公里机井"）、属性条件查询、**自定义 SQL 查询**（高级用户，需做 SQL 注入/只读白名单防护）。
- 展示：列表 + 地图双展示（聚合点），导出 Excel/CSV/Shapefile。
- 注意：自定义 SQL 必须限制为只读、限定表、防注入（MVP 可暂不开放或用预设模板）。

**B1.3 水利设施标注（数据采集主入口）**
- 流程：选"添加水利设施"工具 → 地图点选/输入经纬度 → 填信息窗。底图可切换（卫星/矢量/地形）。
- 必填：名称、类型、所在位置（自动取行政区划可改）、经纬度（自动可改）。选填：建设年份、覆盖面积、受益村组、运行状态、责任人/电话、技术参数（井深/水泵功率/出水量）、维护记录、备注。附件：照片（多张）/图纸/验收报告。
- 规则：生成设施 ID → 标注后进入 **运营人员审核**：通过则前端可见，不通过退回修改。支持 Excel 批量导入（含经纬度，自动解析生成点位）。
- 可测试点：审核状态机（待审核→通过/退回）；批量导入坐标解析。

**B1.4 水利设施修改**
- 入口：列表编辑 / 地图点选编辑。可改全部字段；必填 **修改原因**，记录修改人/时间；保留历史版本。
- 规则：**关键信息（设施类型、经纬度）修改需审批**，改后进入待审核，通过才生效。支持批量修改（如统一变更责任人，需预览）。
- 可测试点：历史版本累积；关键字段触发审批流。

**B1.5 水利设施删除**
- 安全机制：二次确认（"删除后数据不可恢复"提示）+ 必填删除原因（已拆除/重复录入/位置错误）+ **权限控制（仅高级管理员）**。
- 软删除：标"已删除"入回收站，可查看/恢复，**保留 90 天**后物理删除。
- 关联策略：关联业务数据（灌溉/维护记录）默认 **保留**（设施显"已删除"），另可选级联删除。
- 可测试点：软删除不物理删除；回收站恢复；普通用户无删除权限被拦截。

#### B2. 配套设施管理（烘干中心/仓储/育苗/农资供应/交易市场/农机维修/冷链）

**B2.1 列表**：字段 = 名称、类型、位置、经纬度、服务范围（行政区划）、服务能力（如日处理能力/库容/覆盖农户数）、运营状态（正常/停业/建设中）、运营主体（企业/合作社/个体户）、电话、营业时间、资质认证、备注。分页/排序/筛选与水利一致。

**B2.2 查询**：空间/属性/自定义 + 特色：按服务能力筛（"日处理100吨以上烘干中心"）、按服务半径（GIS 网络分析服务覆盖村庄）、按资质认证筛。

**B2.3 标注**：流程同 B1.3，特色：**服务范围绘制**（面状区域，自动算覆盖村庄/农户数）、按类型填差异化服务能力指标、资质证照上传（营业执照/食品经营许可证等）。

**B2.4 修改**：同 B1.4，特色：服务范围调整（重算村庄数，留存变化记录）、服务能力更新（填更新说明）。

**B2.5 删除**：同 B1.5（软删除 + 回收站）。

**B2.6 配套设施分类（基础配置，影响前端所有分类下拉）**
- 新增：分类名称、图标（库选/上传）、排序权重（小靠前）、状态（显示/隐藏，隐藏端不显但管理端可用）。
- 编辑：改名称/图标/排序/状态；状态改隐藏且分类下有设施时提示"该分类下已有 X 个设施，隐藏后不在前端显示"。
- 删除：分类下有设施则阻止，提示"请先转移或删除这些设施"，支持批量转移后删。
- 结构：**两级分类**（一级如"产后处理设施"，二级如"烘干中心/仓储/冷链节点"）。
- 可测试点：删除/隐藏前的关联计数与阻断；两级树正确。

### C. 耕地利用管理

> 连接"地"与"产"的桥梁。种植记录与质量评价均以 **地块编码** 关联确权地块，形成时间序列。

#### C1. 地块种植作物（地块级"种植日记"）

**C1.1 种植数据导入**
- 来源：遥感解译 / 农业部门统计报表 / 农户自主填报 / 网格员巡查（字段含数据来源标识）。
- 核心字段：地块编码（必需，关联确权）、种植年度、种植季节（春/夏/秋）、作物品种、种植面积、播种日期、预计/实际收获日期、产量（公斤/亩，可选）、数据来源、填报人、填报日期。
- 流程：选模板（Excel/CSV/GeoJSON）→ 校验（编码有效性、年度合理性[非未来]、必填完整、面积≤确权面积否则预警）→ 空间关联（带几何按位置关联，否则按编码）→ 入库并按年度分层生成图层。质量报告含失败原因分类统计。
- 可测试点：面积超确权预警；无效编码标记；年度非法拦截。

**C1.2 种植数据修改**
- 方式：字段级修改 / 删除重录。
- 权限约束：农户填报数据本人限时（如播种后 30 天内）自改，超时需审核；遥感数据仅管理员改；改需填原因记人/时间。
- 关键约束：**已收获产量原则不可改**；品种改需佐证；面积改需审批。
- 可测试点：时间窗权限；产量修改拦截。

**C1.3 种植数据删除**：填原因 + 操作日志 + 软删除/回收站。删除影响年度统计与历史，**系统提示优先用"标记为无效"而非删除**。

**C1.4 种植数据查询**：按年度（多选跨年对比）/作物（多选）/五级区划/地块（编码或承包方）/数据来源/产量范围/种植时间。列表 + 地图（不同色块区分作物），导出 Excel/Shapefile。

**C1.5 种植历史**
- 选地块 → 时间轴展示历年每季记录。内容：年度卡片（作物/面积/产量/备注，按作物着色）、**轮作模式视图**（色块条如"稻→稻→稻"连作 或"稻→麦→豆"轮作）、产量趋势折线图。
- 横向对比：多地块同年对比 / 同地块不同年对比。
- 价值：识别长期撂荒、不合理连作、产量下降地块。
- 可测试点：轮作序列拼装；产量趋势数据点正确。

#### C2. 耕地质量评价（地力等级 1–10 等，参照 GB/T 33469）

**C2.1 质量评定列表**：以地块为单元展示最新年度评价。字段：地块编码、坐落、承包方、评价年度、地力等级（颜色区分，绿优红差）、综合得分（百分制）、土壤类型（黑土/棕壤/潮土/砂土）、有机质(g/kg)、全氮(g/kg)、有效磷(mg/kg)、速效钾(mg/kg)、pH、障碍因素（盐碱/瘠薄/渍涝/沙化）、适宜作物、评价机构、报告附件。筛选：年度/等级范围/土壤类型/pH范围/障碍因素/区划。导出 Excel/Shapefile。

**C2.2 质量查询**：属性查询（编码/承包方/区划/年度/等级/土壤/pH/有机质/障碍因素）+ 空间查询（地图绘范围，分层设色展示等级分布）。

**C2.3 质量评价批量维护**：Excel 模板导入（按编码匹配，含指标范围合理性校验如有机质非负）；批量编辑（空间/条件选地块统一改指标，改前预览）。

**C2.4 质量评价删除**：填原因 + 日志 + 权限（仅高级管理员）+ 软删除/回收站。

### D. 撂荒管理 ⭐（平台核心，UI 参考图所示模块）

> 闭环管理：发现 → 核查确认 → 原因分析 → 任务下达 → 治理验收。状态机贯穿全程。

#### D1. 撂荒地块管理

**D1.1 撂荒地块增加（3 种来源）**
- **途径一·疑似图斑确认入库**：遥感识别生成疑似图斑（编号/位置/面积/识别依据/识别时间/影像时相）→ 下发网格员现场核查（GPS 签到 + 拍照 + 核查结论）→ 确认则转正为撂荒记录，补充：撂荒面积（自动算可改）、撂荒起始时间、撂荒程度（完全/部分）、现场照片。
- **途径二·网格员巡查直接上报**：地图定位 + 绘边界 + 填信息（坐落/面积/起始时间/程度）+ 拍照 + 发现人 → 状态"待确认"，上级审核后入库。
- **途径三·群众举报转化**：运营人员核实后将举报线索转化，关联举报记录 + 核实说明。
- **关联确权**：空间自动匹配确权地块编码；仅覆盖部分时支持 **部分关联**，记录撂荒面积占该确权地块比例。
- 可测试点：三途径各自状态初值（待确认/待核查）；部分关联占比计算。

**D1.2 撂荒地块修改**
- 基本信息：边界调整（重算面积）、起始时间修正、程度调整。
- 治理信息：治理状态变更（未治理→治理中→已治理）、治理措施记录、治理责任人。
- 原因补充：撂荒原因分类与说明。
- 规则：必填修改原因 + 记人/时间 + 历史版本；**边界调整致面积变化 >10% 需审批**。

**D1.3 撂荒地块删除**
- 场景：误判删除（实为耕种，遥感误判）/ 重复删除（多源重复入库）/ 关联错误删除。
- 流程：选删除原因（误判/重复/关联错误/其他）+ 详细说明 + 软删除/回收站 90 天。
- **约束：已下发治理任务且治理中的记录，需先终止治理任务才能删。**
- 可测试点：治理中记录删除被阻断。

**D1.4 撂荒地块查询**：按区划 / 地块属性（编码/承包方）/ 撂荒特征（面积范围、起始时间范围、程度、持续时间如"3年以上"）/ **治理状态（未治理/治理中/已治理，核心筛选）** / 原因类型（劳动力/经济/基础设施/土壤/灾害/流转）/ 来源（遥感/巡查/举报）/ 空间范围。列表 + 地图（红色斑块），导出 Excel/CSV/Shapefile（撂荒治理作战图）。

#### D2. 撂荒原因填报

**撂荒原因分类体系（标准化预设）：**
- A 劳动力因素：劳动力不足、人口老龄化
- B 经济因素：种植效益低、生产成本高、农产品价格波动大
- C 基础设施因素：水利设施缺失/损毁、田间道路不通、耕地碎片化
- D 土壤因素：土壤贫瘠、盐碱化、水土流失、污染
- E 自然灾害因素：洪涝、干旱、病虫害
- F 土地流转因素：流转不畅、流转价格低、流转纠纷
- G 其他：征占地待开发、重点项目影响

**D2.1 撂荒原因上报**：选原因（可多选 + 具体说明）+ 分类别填补充信息（劳动力→外出务工/留守老人数；经济→亩均投入/产出）+ 上传佐证 + 填建议措施（如"建议修复东沟泵站"）。

**D2.2 撂荒原因查询**：按区划、原因类型、填报时间检索。

#### D3. 撂荒治理任务（流程化、闭环化）

**D3.1 治理任务下发**：运营人员筛选"未治理"地块 → 批量建任务下发。填：任务基本信息（名称可自动生成"XX村撂荒地治理任务20240301"、描述、下发时间、完成时限）、责任单位与责任人（乡镇/村 + 村书记/主任/网格长）、治理目标（面积目标/治理标准/质量要求）、配套支持（种子/农机补贴/水利修复）、附件。下发后责任人收通知（站内信/短信/微信）。

**D3.2 治理任务办理**：责任单位制定治理方案（思路/措施/步骤时间节点/所需支持，提交后上级可审核提意见）；治理进度跟踪（按阶段填报百分比进度条 + 阶段佐证，自动生成进度曲线对比计划与实际偏差）；问题反馈（在线反馈，上级响应协调）。

**D3.3 治理验收反馈**：责任单位提交完成确认（完成总结 + 治理后照片 + **关联新增种植数据**[选治理后作物/面积] + 待验收事项）→ 上级组织验收（线上审核 + 线下实地抽查）→ 通过则撂荒状态更新"已治理"并移出待治理列表；不通过反馈原因退回办理整改。

**D3.4 治理数据下载**：导出 Excel/PDF。内容：撂荒地块清单、治理任务台账、治理进度报表、治理成效汇总（按区划汇总治理面积/治理率）。

> **状态机汇总（撂荒地块）**：`待确认/待核查 → 未治理 → 治理中 → 已治理`；治理任务：`已下发 → 方案制定 → 办理中 → 待验收 → 验收通过/退回`。

### E. 种植动态分析

> 从数据管理走向分析、预测、决策。**重计算与可视化，MVP 可先做 E1.1/E1.2/E1.3/E1.5 的统计图表，E1.4/E1.6/E2 视进度。**

#### E1. 数据分析

- **E1.1 行政区划分析**：按省/市/区县/乡镇/村组树形下钻。区域种植概况报告：种植面积汇总及占比、种植结构饼图、产量统计（总产/单产 + 与全县市对比）、近3-5年趋势折线、排名（在上级区划中排名）。支持多区域横向对比。
- **E1.2 年度变化分析**：选时段（如 2020–2024）。面积变化趋势折线、变化显著性识别（增>20% 红上箭头 / 降>20% 绿下箭头）、结构变化桑基图、变化归因分析（结合政策/价格）。
- **E1.3 种植结构分析**：饼/环图、雷达图、多区域对比图 + **种植结构合理性评价**（如"水稻占比>70% 结构单一有病虫害风险"）。
- **E1.4 优势产区识别**：单作物适宜性评价（光温水土，0–100 分）+ 优势度计算（近3-5年平均产量/稳定性/品质）+ 空间聚类（连片高适宜）。热力图展示，推荐优势产区名单与规模建议。
- **E1.5 耕地利用类型**：分类 = 在耕 / 季节性闲置 / 撂荒 / 非农化。基于遥感 + 种植数据自动判定，生成分布专题图 + 占比饼图。监测"非粮化/非农化"与撂荒治理成效。
- **E1.6 时间轴动态演变**：多年种植/撂荒/耕地利用变化按时间动态呈现，拖动时间轴地图色块逐年变化，支持播放/暂停/逐帧/倍速。

#### E2. 数字化地图服务（面向农业企业）

- **E2.1 作物种植空间分布**：选作物品种 + 年度 → 地图按色块绘种植分布，支持多作物叠加、点击查详情、按区划/自定义范围裁剪下载（JPG/PNG/PDF）。
- **E2.2 产地分布图层**：预定义特色农产品产地（延吉大米、苹果梨等）专题图，主产区/核心区/辐射区分层设色 + 生产基地点位标注。
- **E2.3 产地详情查询**：点击产地聚合展示基础/生产/品质/供应链多维信息，支持产销对接联系。
- **E2.4 产地供应链**：流程/网络图展示从投入品→生产→加工→仓储物流→销售全链条，支持上溯下追。

**测试种子数据要求（参考成熟农业/政务项目）：**
- 行政区划：至少 1 省 / 2 市 / 4 区县 / 若干乡镇村组，构成可下钻的真实树。
- 确权地块：≥ 50 块，覆盖多村组、多用途、多承包方，含面积差异样本。
- 设施：水利每类型 ≥ 3 条（含正常/维修/废弃/待改造各状态）；配套覆盖各分类。

- 撂荒：覆盖三种来源、各治理状态、各原因类型（对应 UI 参考图：待审核/已治理/治理中/未治理/已驳回），含部分关联样本。
- 质量评价：覆盖 1–10 各地力等级、不同土壤类型与障碍因素。

> 字段命名、状态枚举、软删除/审计等模式参考主流后台脚手架（如 RuoYi / pig 等）的工程化约定，保证规整、可维护、可测试。

## 7. UI 规范（参考所给截图风格）

- **整体布局**：左侧深色竖向图标导航（首页/地图/地块/设施/种植/撂荒/分析/质量/用户）+ 顶部面包屑 + 多标签页（Tab）+ 右上角通知/全屏/用户。
- **列表页统一范式**（参考撂荒管理截图）：顶部筛选栏（输入框 + 下拉 + 日期 + 搜索按钮 + 导出 Excel + 统计分析 + 右侧新增按钮）→ 数据表格（多选列 + 业务列 + 状态标签 Tag + 操作列：任务/编辑/删除）→ 底部分页（Total + 页码）。

- **组件**：Element Plus `el-table` / `el-form` / `el-pagination` / `el-tag` / `el-dialog`（编辑/删除二次确认）/ `el-tabs`；图表用 ECharts（饼/折线/雷达/桑基/热力）。
- **一致性**：所有列表/表单/弹窗复用统一封装组件（搜索栏、表格、分页、上传、地图选点），保证规整美观与可维护。
